## 我的世界（World）模块设计规范

### 模块定位

「我的世界」是 Odyssey 的核心模块，也是整个产品最重要的页面。用户进入 Odyssey 后首先看到的应该是自己建设中的文明世界，而不是传统学习平台中的统计数据、技能分数和图表。Odyssey 的核心理念是“学习推动文明发展”，因此用户每一次学习、完成任务、获得技能成长，都应该直接反映到世界地图、建筑升级、文明扩张和时代演进之中。

### 页面目标

当用户打开“我的世界”页面时，应该产生一种正在经营和建设个人文明的感觉，而不是查看后台管理系统。页面需要让用户清晰看到自己的文明等级、当前时代、已建造建筑、资源储备、探索进度以及未来发展方向，从而形成持续成长的动力。

### 页面布局

页面采用三层结构。顶部为文明概览区域，中间为文明地图区域，底部为分析与成长区域。

顶部区域展示当前文明等级、文明指数、知识点、科技点、人口数量、探索进度以及当前时代。例如：

* 文明等级：Lv42
* 当前时代：智能时代 III
* 文明指数：8680 / 12000
* 人口：128450
* 知识点：12450
* 科技点：8230
* 探索进度：68%

这些数据采用统一视觉组件展示，而不是多个独立统计卡片。

### 文明地图

文明地图是页面的核心视觉区域，占据首屏大部分空间。地图采用俯视角设计，整体风格偏向温暖的数字文明世界，而非传统游戏地图。

地图中央为主城，代表用户当前文明等级。随着文明升级，主城会从木屋逐渐演化为学院、城市、都市、文明中心以及最终的世界奇观。

地图周围分布不同文明建筑，例如：

* 知识殿堂
* 文明图书馆
* 自动化工厂
* AI议会
* 智能研究院
* 商业中心
* 数字银行
* 设计学院
* 训练场
* 科学研究院

每个建筑都与技能体系直接关联。建筑等级越高，视觉表现越复杂、规模越宏伟。

### 建筑成长机制

建筑采用十级成长体系。

Lv1 为基础设施，例如木屋、小工坊、小书屋。

Lv2~Lv4 为区域建筑，例如学院、车间、实验室。

Lv5~Lv7 为大型文明建筑，例如自动化工厂、知识殿堂、商业中心。

Lv8~Lv10 为世界级建筑，例如文明图书馆、AI议会、数字文明主脑以及文明奇观。

用户通过提升相关技能获得建筑升级资格。

例如：

自动化工厂需要 Workflow、Python 和 Agent 技能；

知识殿堂需要 Reading、Research 和 RAG 技能；

数字银行需要 Finance、Data 和 Coding 技能；

智慧城市需要 Management、AI、Engineering 和 Data 技能。

### 世界探索系统

地图边缘存在迷雾区域。迷雾区域代表尚未探索的知识领域和文明区域。用户达到特定技能要求后可以解锁新的区域。

例如：

* Prompt Engineering 达到 50 解锁 AI 峡谷；
* Workflow 达到 70 解锁 工业平原；
* Research 达到 80 解锁 知识高原；
* Design 达到 60 解锁 创意群岛；
* Business 达到 90 解锁 贸易港湾。

新区域会带来新的建筑、资源和任务。

### 世界事件系统

系统定期触发文明事件，用于增强成长反馈和世界活力。

典型事件包括：

* 知识大爆发
* 科技革命
* 商业繁荣
* 文化交流
* 资源发现
* 文明庆典

事件完成后可获得知识点、科技点、文明值、建筑加速卡以及特殊徽章等奖励。

### 页面视觉设计

整体风格参考 Civilization、SimCity、Notion 和 Linear 的结合体。采用奶油白、鼠尾草绿、暖灰色和浅金色作为主要配色，避免蓝紫色科技风和传统后台风格。

界面使用大圆角、细描边、柔和阴影和 Bento Grid 布局。文明地图作为视觉中心，其余模块围绕地图组织。所有视觉设计都应围绕“成长、探索、建设、文明”四个关键词展开。

### Claude 开发要求

设计必须基于真实前端技术可实现原则。禁止使用纯概念插画式设计和无法落地的游戏 CG 场景。优先使用项目已有技术栈，推荐使用 React、Tailwind、Shadcn、SVG、Canvas、ECharts 等技术实现。文明地图可考虑采用 SVG 节点地图或 Canvas 地图方案，每个建筑均为独立可交互节点，支持点击查看详情、升级动画、建造状态和成长记录。

最终目标是让用户打开 Odyssey 时看到的是一个不断成长的文明世界，而不是一个学习数据面板。
