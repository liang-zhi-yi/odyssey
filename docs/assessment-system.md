# Odyssey Assessment System v1
## Overview
### Odyssey 不以学习时长、课程完成度、打卡次数衡量成长。
### Odyssey 以能力掌握度衡量成长。
### 每个 Skill 都通过统一的四维能力模型进行评估。
---
## Four-Dimension Capability Model
### 每项技能拆分为：
## Knowledge
### 知道什么
### 评估用户是否掌握相关概念、术语和基础知识。
---
## Reasoning
### 理解为什么
### 评估用户是否理解底层原理、设计思路和决策逻辑。
---
## Application
### 解决问题
### 评估用户是否能够将知识应用到真实场景。
---
## Creation
### 创造能力
### 评估用户是否能够设计完整方案、架构或项目。
---
## Score Calculation
### 每个维度：
### 0~100
### 综合评分：
#### Overall Score
#### =
#### Knowledge × 20%
#### +
#### Reasoning × 25%
#### +
#### Application × 35%
#### +
#### Creation × 20%
### 原因：
    Application 最重要。
    知道概念不代表会用。
---
# Prompt Engineering Assessment
---
## Prompt Novice
### 0~20
### 能力特征：
- 知道 Prompt 是什么
- 知道 Role Prompt
- 知道 Few-shot
- 会写简单提示词
### 验证方式：
### Knowledge：
    什么是 Prompt？
    什么是 Few-shot？
    什么是 Role Prompt？

## Application：
    写一个翻译 Prompt
    Prompt Beginner
---
## Prompt Beginner
### 20~40
### 能力特征：
- 能设计结构化 Prompt
- 能使用 Few-shot
- 能控制输出格式
### 验证方式：
### Application：
    设计一个用户评论分类 Prompt
### Reasoning：
    为什么 Few-shot 能提高效果？
    Prompt Practitioner
---
## Prompt Practitioner
### 40~60
### 能力特征：
- 能解决真实业务问题
- 能优化 Prompt
- 能发现 Prompt 缺陷
### Quest 示例：
    运营团队希望自动分类用户反馈
    设计完整 Prompt

### 评估项：
- 准确率
- 稳定性
- 鲁棒性
---
## Prompt Engineer
### 60~80
### 能力特征：
- 能设计复杂 Prompt System
- 能处理长上下文
- 能处理结构化任务
### Quest 示例：
    设计客服 AI Prompt Framework
### 要求：
- 情绪安抚
- FAQ回答
- 意图识别
- JSON输出
---
## Prompt Architect
### 80~100
### 能力特征：
- 设计企业级 Prompt Framework
- 形成可复用 Prompt Pattern
### Quest 示例：
    设计完整 Agent Prompt Layer
### 包括：
- Planner Prompt
- Executor Prompt
- Reviewer Prompt
- Memory Prompt
---

# Assessment Philosophy
### Odyssey 不关心：
### 学习了多久
### 不关心：
### 看了多少课程
### 不关心：
### 做了多少笔记
### Odyssey 只关心：
### 你是否证明了自己拥有这项能力
